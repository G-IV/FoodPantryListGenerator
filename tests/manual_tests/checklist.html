FoodPantryListGenerator — Manual Test Checklist

Branch: feat/invalid-barcode-alert  |  Print this page or work through it on screen.

Before you start — one-time setup on the Surface Pro
  1. Copy tests\fixtures\InvNmbrs.csv to C:\DoubleCheck\InvNmbrs.csv (replace any existing file).
    This gives you the full set of flagged case numbers from the test fixture so every scenario below produces a predictable result.
  2. Open tests\manual_tests\test_barcodes.html in a browser and print it, or keep it on screen to scan from.
  3. Start the application: double-click FoodPantry ListGenerator on the desktop.

1 — Startup display

  • 1.1 Output filename is shown and contains today's date (e.g. scanned_barcodes20260505.csv).
  • 1.2 Record count is shown (Records already in file: 0 for a fresh session).
  • 1.3 Release URL is shown: Recent release: https://github.com/G-IV/FoodPantryListGenerator/releases/latest
  • 1.4 Prompt reads Scan a barcode, or press Enter to exit.

2 — Normal scan (clean case number — C11xxxxx range)

  • 2.1 Scan any C11xxxxx barcode from the test sheet.
    These are NOT in InvNmbrs.csv — they should be accepted normally.
  • 2.2 A green ✓ checkmark appears after the scan.
  • 2.3 The record count increments on the next prompt.
  • 2.4 After exiting, confirm the case number appears in the output CSV at C:\DoubleCheck\.

3 — Flagged scan (case number in InvNmbrs.csv — C12xxxxx or higher range)

  • 3.1 Scan any C12xxxxx barcode from the test sheet.
    These ARE in InvNmbrs.csv — an alert banner should appear.
  • 3.2 A red ✗ and an ANSI alert banner are displayed.
  • 3.3 The banner shows the escort message and does not contain any contact name or phone number.
  • 3.4 The flagged case number does not appear in the output CSV after exiting.
  • 3.5 A file named flagged_barcodes20YYMMDD.csv (today's date) exists in C:\DoubleCheck\ after the flagged scan.
    Open it and confirm it contains a row with the case number and a timestamp. It should have exactly two columns — no empty merge fields.
  • 3.6 Scan a second flagged barcode. Confirm the new row is appended to the existing file rather than overwriting it.
  • 3.7 Repeat with one barcode from the C13xxxxx, C14xxxxx, and C15xxxxx ranges to confirm all flagged ranges alert correctly.

4 — Manual entry (volunteer types a case number)

  • 4.1 At the scan prompt, type 1100099 (no C prefix) and press Enter.
    Simulates a volunteer manually keying in a number when the card won't scan.
  • 4.2 App displays C1100099 with a green ✓.
  • 4.3 Type C1100099 (with C prefix) and confirm the app handles it the same way.

5 — InvNmbrs.csv auto-creation

Close the app, then delete C:\DoubleCheck\InvNmbrs.csv, then restart.

  • 5.1 The on-screen notice appears:
    NOTE: InvNmbrs.csv was not found and has been created automatically.
  • 5.2 C:\DoubleCheck\InvNmbrs.csv now exists and contains exactly two lines: the contact row and the Case # header.
  • 5.3 No barcodes are flagged (the new file has no case numbers in it).
  • 5.4 Restore InvNmbrs.csv from the fixture before continuing: copy tests\fixtures\InvNmbrs.csv back to C:\DoubleCheck\.

6 — InvNmbrs.csv format validation

Add a malformed row to C:\DoubleCheck\InvNmbrs.csv (e.g. append the line BADROW), then restart the app.

  • 6.1 The app starts without crashing.
  • 6.2 BADROW no longer appears in InvNmbrs.csv after startup.
  • 6.3 InvNmbrs_errors.log exists in C:\DoubleCheck\ and records the removed row.
  • 6.4 Restore InvNmbrs.csv from the fixture before continuing.

7 — Exit

  • 7.1 Press Enter on a blank scan prompt.
  • 7.2 App prints Barcodes saved to: scanned_barcodes…csv and closes cleanly.
