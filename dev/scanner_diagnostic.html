Scanner Diagnostic

This page helps determine whether the Tera scanner adds a prefix and suffix to scanned barcodes, or whether those characters are encoded in the barcode itself. This affects how the application parses barcode input.

Step 1 — Start the application in diagnostic mode

In your terminal, with the virtual environment active, run the application from the dev/output/ directory so test CSV output stays separate: 

cd dev/output
python ../../FoodPantryListGenerator.py

Leave the terminal visible so you can read what gets printed after each scan.

Step 2 — Scan the test barcodes below

Each barcode encodes a different string. Scan each one and note exactly what the application prints on screen after each scan. Record the results in Step 3.

Hold the scanner 4–8 inches from the screen. The barcodes can be scanned directly from this page — you do not need to print them.

Barcode A — encodes: 12345

Barcode B — encodes: 01052089

Step 3 — Record what the application printed

After scanning each barcode, look at what the application printed on the line starting with "Record N — Please scan a barcode:" on the NEXT prompt. The app prints the case number it parsed before asking for the next scan.

Compare your results to the table below:

Barcode Encoded value If app shows C12345 or C1052089 If app shows something else
A 12345 Scanner does NOT add prefix/suffix.
The {[C]…} wrapper is encoded in the real customer barcodes themselves.		 Note exactly what was printed and share it with the developer.
B 01052089 Same conclusion as above.
Leading zeros are handled correctly.		 Note exactly what was printed and share it with the developer.

Step 4 — Exit the application

Press Enter on a blank scan prompt to close the program.

A CSV file will have been created in dev/output/. You can open it to see the raw output rows alongside the case numbers. This file is gitignored and will not be committed.

After the diagnostic

Share your results with the developer. The outcome determines whether the real customer barcodes should be added to dev/test_barcodes.html as plain numbers, or as the full {[C]NNNNN} encoded string.
