Test Barcodes — FoodPantryListGenerator Dev Tool

These barcodes simulate real customer ID cards for development and testing. Each one encodes the full string that a real card would contain, including the {[C]…} wrapper that the customer ID card barcodes use.

How to use: Start the application from dev/output/ so test CSV files stay out of the main directory: 
cd dev/output
python ../../FoodPantryListGenerator.py
Then scan barcodes from this page. The Expected output listed under each barcode is what the application should display after scanning.

Standard case numbers (scanner format)

These simulate real customer ID cards. The encoded string includes the {[C]…} wrapper.

Encodes: {[C]01052089}
Standard case — leading zero
Expected: C1052089
Encodes: {[C]00012345}
Multiple leading zeros
Expected: C12345
Encodes: {[C]9876543}
No leading zeros
Expected: C9876543
Encodes: {[C]00000001}
Extreme leading zeros
Expected: C1

Manual entry simulation

These simulate what a volunteer types when a card won't scan. They encode plain numbers only — no {[C]…} wrapper. The application should recognise these as manual entry and handle them correctly.

Encodes: 1052089
Plain number — no prefix
Expected: C1052089
Encodes: 0012345
Plain number — leading zeros
